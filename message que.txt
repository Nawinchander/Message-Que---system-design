A Message Queue is a system design component that allows different services to communicate asynchronously.

👉 Instead of calling directly (like REST API),
you send a message to a queue, and another service processes it later.



Simple Analogy

Think of it like Swiggy/Zomato orders:

You place order → goes to queue
Restaurant picks it when ready
Delivery partner handles it later

👉 You don’t wait → system is decoupled + scalable

🔥 Why use Message Queue?
Without MQ (Bad)


User → API → Email Service → Payment → Notification


If email fails → whole system fails
❌ Slow response


User → API → Queue → Worker Services

✅ Fast response
✅ Retry possible
✅ Scalable
✅ Fault tolerant
